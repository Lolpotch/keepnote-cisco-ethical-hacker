PART 3: RECON-NG
STEP 1: CREATE A WORKSPACE

└─$ recon-ng
# Starts recon-ng engine.

Usage: workspaces <create|list|load|remove> [...]

[recon-ng][default] > workspaces list
# Show list of workspaces in recon-ng.

[recon-ng][default] > workspaces create cisco
# Create a workspace named "cisco"

STEP 2&3: INVESTIGATE MODULES

[recon-ng][cisco] > marketplace search
# Show list of marketplace.

[recon-ng][cisco] > marketplace info <module-path>
# More info on the module.

STEP 4&5: INSTALL & RUN MODULES

[recon-ng][cisco] > marketplace install recon/domains-hosts/hackertarget
# Install the module.

[recon-ng][cisco] > modules load hackertarget
# Load the module.

[recon-ng][cisco][hackertarget] > info
# Shows module's name, version, developer, brief desc., and options.

Usage: options set <option> <value>

[recon-ng][cisco][hackertarget] > options set source hackxor.net
# "options set" = command
# "source" = option
# "hackxor.net" = target

# Run info command again to verify the option update.

[recon-ng][cisco][hackertarget] > run
# Runs the module.

[recon-ng][cisco][hackertarget] > dashboard
# Shows the gathered information

Usage: show <companies|contacts|credentials|domains|hosts|leaks|locations|netblocks|ports|profiles|pushpins|repositories|vulnerabilities>

[recon-ng][cisco][hackertarget] > show hosts
# Show discovered hosts (subdomains)

STEP 6: RECON-NG WEB INTERFACE

└─$ recon-web
# Show results in GUI via browse
